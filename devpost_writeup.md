# Devpost Write-up: Next-Gen Multimodal AI Language Tutor

## Inspiration
Learning a language requires speaking it. Traditional language learning apps are excellent for vocabulary and grammar, but they fail to provide the most crucial element: real-time, unstructured conversational practice. Hiring a human tutor is expensive and hard to schedule. We wanted to build an AI tutor that feels truly alive—one that sees what you show it, hears your pronunciation, and gracefully handles your interruptions, just like a real teacher.

## What it does
The Next-Gen Multimodal AI Language Tutor provides structured 15-minute language classes driven by the **Gemini 2.5 Live API**.
Users start by logging in (anonymously via Firebase) and selecting a structured lesson (e.g., "Ordering Food" or "Job Interview Practice", sorted by CEFR levels). 
Upon joining the classroom, the LiveKit engine connects the user to a dedicated Node.js-based Gemini agent.
The user speaks naturally. If the agent is speaking and the user interrupts, the agent instantly stops talking and listens—a feature made possible by LiveKit's WebRTC integration.
Furthermore, the user can click "Show Teacher" to stream a 1fps live video feed. This allows the user to hold up objects (e.g., "What is the Spanish word for this?") or written text, and the Gemini multimodal model will seamlessly incorporate that visual context into the conversation.

## How we built it
- **Architectural Backbone:** We separated the logic into a React frontend, a Node.js Express API, and a LiveKit Node Agent process.
- **Frontend (React + Vite + Tailwind):** We utilized `@livekit/components-react` to rapidly build the classroom UI, handling the WebRTC audio tracks, video tracks, and stunning agent state visualizations (speaking/listening). We used Firebase Auth to securely issue JWTs.
- **Backend (Node.js + PostgreSQL):** Hosted on Google Cloud Run. It validates the Firebase JWTs, queries a Cloud SQL PostgreSQL database for lesson contexts, and dynamically issues LiveKit connection tokens.
- **The Brain (LiveKit Agents + Gemini):** We utilized the brand-new `@livekit/agents-plugin-google` for Node.js. This allowed us to instantiate the Gemini 2.5 flash model with specific system prompts pulled from the database, giving the conversational AI deep context about the current pedagogy goals.
- **Infrastructure:** We automated the GCP resource provisioning (Cloud SQL, Cloud Run, Secret Manager) using Terraform.

## Challenges we ran into
- WebRTC video ingestion can easily overwhelm LLMs. We had to ensure the camera feed was throttled/sampled appropriately to a maximum of 1 frame per second so Gemini could ingest the visual context without token overflow.
- Managing states securely: ensuring only authenticated users could receive LiveKit tokens via the JWT validation process took careful orchestration.

## Accomplishments that we're proud of
- Achieving genuine conversational latency (< 500ms). The tutor truly feels present.
- The barge-in capability works flawlessly, making the conversations feel human rather than transactional.

## What's next
- Expanding the curriculum to include 50+ language pairs.
- Adding a post-lesson summary dashboard generated by Gemini, returning a score from 0-100 based on grammatical accuracy, stored back into the PostgreSQL `Metrics` table.
